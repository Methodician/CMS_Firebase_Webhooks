Steps I took:

Get things started:

1) Signed up for butter trial with methodician

2) PW was Test!23

3) Created new project with Angular CLI, specifying that I wanted routing, didn't want tests, and wanted "hook" as my prefix

4) npm i firebase @angular/fire buttercms

5) npm i @angular/material @angular/cdk @angular/animations (Optional step from buter quickstart)

6) Used styles and themes from Flight since I'm not great with css/scss



Build out basics:

7) Created blog (detail/list/preview), home, admin and nav-menu components (ng g component components/blog)

8) Created a blog/butter service (ng g service services/blog)

9) Added all the relevant routes to my nav and routing module

10) Set up my blog components and service to work with my headless CMS (Butter)

11) Used Flight's cool carousel system to display featured posts, sompe pipes for HTML, etc...



Introduce functions:

1) in the terminal: firebase init... I selected RTDB, Functions, and Hosting and chose my default project and defaults except TypeScript for Functions and declined TSLint (since this is a quick demo and I've had issues with it in the past) and I chose "dist" as my public directory and "yes" to configure as SPA since this is an Angular project after all...

3) Opened a terminal to the Functions folder and npm install buttercms

4) Import admin and Butter into functions/src/index.ts

5) Create my butter constant just like in BlogService

6) Created my https post Firebase cloud functions to plug webhooks into

7) in the terminal: firebase deploy --only funcitons

Hook in Webhooks

1) Open Firebase console to your project, open Functions from left panel, find your trigger url for the new function, and copy it

2) Open ButterCMS dashboard. Click your name in upper left --> Settings --> Webhooks and paste the copied url into "Target URL" and select "Blog Post Create, Published, Update, or Delete" from the "Event" dropdown

3) Try creating, updating, and deleting posts in the CMS to make sure it's working. Look at the Functions log, look at the database, is all good?

Other things you could do:

Butter doesn't return a compressed version of your posts or pages. You could use your Firebase list of slugs and/or some condensed preview objects if Butter's return was too massive (special cases)



You could use Firebase to create comment system or chats specific to posts or pages.
