Steps I took:

Get things started:

1) Signed up for butter trial with methodician

2) PW was Test!23

3) Created new project with Angular CLI, specifying that I wanted routing, didn't want tests, and wanted "hook" as my prefix

4) npm i firebase @angular/fire buttercms

5) npm i @angular/material @angular/cdk @angular/animations (Optional step from buter quickstart)

6) Used styles and themes from Flight since I'm not great with css/scss

Build out basics:

7) Created blog (detail/list/preview), home, admin and nav-menu components (ng g component components/blog)

8) Created a blog/butter service (ng g service services/blog)

9) Added all the relevant routes to my nav and routing module

10) Set up my blog components and service to work with my headless CMS (Butter)

11) Used Flight's cool carousel system to display featured posts, sompe pipes for HTML, etc...

Introduce functions:

1) in the terminal: firebase init... I selected RTDB, Functions, and Hosting and chose my default project and defaults except TypeScript for Functions and declined TSLint (since this is a quick demo and I've had issues with it in the past) and I chose "dist" as my public directory and "yes" to configure as SPA since this is an Angular project after all...
